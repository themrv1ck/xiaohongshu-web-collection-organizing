# xiaohongshu-web-collection-organizing

小红书网页端收藏整理工作流（macOS + Chrome + AppleScript + DOM 注入）。

## 这是什么

这是一个面向他人复用的实战文档，核心目标是：

- 读取已登录的小红书网页端收藏
- 提取当前实际可见的收藏标题与链接
- 按主题做分组
- 创建专辑
- 通过 `取消收藏 -> 重新收藏 -> 收藏成功 banner -> 加入专辑 -> 选择目标专辑` 的网页路径，把笔记放进指定专辑

## 适用前提

- macOS
- Google Chrome
- 用户已经登录小红书网页版
- Chrome 已开启：`查看 -> 开发者 -> 允许 Apple 事件中的 JavaScript`

## 当前结论

- 可以读取收藏页和专辑页 DOM
- 可以自动创建专辑
- 可以自动把单条收藏加入指定专辑
- 可以批量执行，但必须接受以下现实：
  - 页面加载慢，必须长等待 + 重试
  - 直接对 `.collect-wrapper` 做 DOM `.click()` 不总是稳定
  - 更稳的是派发鼠标事件序列：`mouseover -> mousedown -> mouseup -> click`
  - 分类绝不能只靠一次粗糙关键词；正式批量执行前要人工复核

## 分类策略的最新经验

不要把 `杂项灵感` 当成长期终点，只能当兜底缓冲区。

如果出现以下情况，说明要先细化再跑：

- `杂项灵感` 数量明显过大
- 用户指出很多内容被错分
- 某些主题本来能单独成专辑，却被塞进大类里

实战中的具体修正例子：

- “挑第一只手表”应归入 `穿搭发型与品味`
- 如果滑雪内容足够多，应单独创建 `滑雪` 专辑，而不是继续放进泛化的 `运动训练与体态`
- 如果用户指出“丢了两个收藏夹/专辑”，必须回到抓取和分类阶段重新核对总量，不要直接继续批处理

## 推荐工作流

1. 打开收藏页并滚动抓取实际可见收藏
2. 导出 `id / title / href / xsec_token / first_seen`
3. 做第一轮语义分类
4. 人工复核分类结果，尤其检查：
   - 杂项是否过大
   - 是否需要新增更细专辑（例如滑雪）
   - 是否有明显误分项目
5. 创建/补齐专辑
6. 对每条笔记执行：
   - `uncollect`
   - 用原始收藏链接进入笔记页
   - 等待 `.collect-wrapper` 挂载
   - 触发重新收藏
   - 点击 banner 的 `加入专辑`
   - 点击目标专辑
7. 跑完后做抽样验证和数量核对
